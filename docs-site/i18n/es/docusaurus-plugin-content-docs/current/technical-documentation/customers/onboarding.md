## Paso 1: CreaciГіn del Cliente

### Desde el Panel de AdministraciГіn

1. Navegar a **Clientes** > **Nuevo Cliente**
2. Completar informaciГіn bГЎsica:
   - Email
   - Telegram ID (opcional)
   - Tipo de cliente
3. Hacer clic en **Crear**

### Datos Requeridos

| Campo | Tipo | Requerido | DescripciГіn |
|-------|------|-----------|-------------|
| email | String | SГӯ | Email Гәnico del cliente |
| telegramHandle | String | No | ID de Telegram para notificaciones |
| customerType | Enum | SГӯ | ClasificaciГіn del cliente |

### Via API GraphQL

```graphql
mutation CreateCustomer($input: CustomerCreateInput!) {
  customerCreate(input: $input) {
    customer {
      id
      publicId
      email
      status
    }
  }
}
```

## Paso 2: VerificaciГіn KYC

### Inicio de la VerificaciГіn

Una vez creado el cliente, se puede iniciar la verificaciГіn KYC:

1. Navegar al detalle del cliente
2. Hacer clic en **Iniciar KYC**
3. Se genera un enlace de verificaciГіn de Sumsub

### Estados de KYC

| Estado | DescripciГіn | Siguiente AcciГіn |
|--------|-------------|------------------|
| NOT_STARTED | KYC no iniciado | Iniciar verificaciГіn |
| PENDING | VerificaciГіn en progreso | Esperar resultado |
| APPROVED | Identidad verificada | Proceder a activaciГіn |
| REJECTED | VerificaciГіn fallida | Revisar y reintentar |
| REVIEW_NEEDED | Requiere revisiГіn manual | Revisar en Sumsub |

### IntegraciГіn con Sumsub

El sistema se integra con Sumsub para verificaciГіn de identidad:

1. **GeneraciГіn de enlace**: Se crea un enlace Гәnico para el cliente
2. **VerificaciГіn**: El cliente completa el proceso en Sumsub
3. **Webhook**: Sumsub notifica el resultado vГӯa webhook
4. **ActualizaciГіn**: El estado del cliente se actualiza automГЎticamente

### Monitoreo del Estado

```graphql
query GetCustomerKycStatus($id: ID!) {
  customer(id: $id) {
    id
    kycStatus
    applicant {
      status
      reviewResult
      createdAt
    }
  }
}
```

## Paso 3: Aprovisionamiento AutomГЎtico

### Usuario en Keycloak

Cuando el KYC es aprobado, automГЎticamente:

1. Se crea un usuario en Keycloak (realm customer)
2. Se envГӯa email de bienvenida con credenciales
3. El cliente puede acceder al portal

### Cuenta de DepГіsito

SimultГЎneamente se crea:

1. Cuenta de depГіsito en el sistema
2. Cuentas contables en el libro mayor
3. RelaciГіn cliente-cuenta establecida

### Eventos Generados

| Evento | Disparado Por | Consecuencia |
|--------|---------------|--------------|
| KycApproved | Webhook Sumsub | Inicia aprovisionamiento |
| UserCreated | Job de onboarding | Usuario listo en Keycloak |
| DepositAccountCreated | Job de onboarding | Cuenta lista para operar |
| CustomerActivated | Completar onboarding | Cliente puede operar |

## GestiГіn de Errores

### KYC Rechazado

Si el KYC es rechazado:

1. Revisar motivo en Sumsub
2. Contactar al cliente si es necesario
3. Reiniciar proceso de verificaciГіn si aplica

### Errores de Aprovisionamiento

Si falla el aprovisionamiento:

1. Verificar logs del sistema
2. Revisar estado de Keycloak
3. Ejecutar job de aprovisionamiento manualmente si es necesario
